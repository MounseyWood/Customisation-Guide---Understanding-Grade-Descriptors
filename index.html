Customisation Guide for Understanding Grade Descriptors Tool

Kingston University London, Kingston School of Art

A guide for faculty to customise the Understanding Grade Descriptors interactive tool for their own programmes

1. Introduction

This guide provides step-by-step instructions for customising the "Understanding Grade Descriptors" interactive tool for your specific programme or module. The tool helps students understand the criteria used for grading their work and provides guidance on how to improve their performance.

Note: This guide assumes basic familiarity with HTML files but does not require advanced coding skills. All customisations can be done using simple text editing.

1.1 Purpose of the Tool

The Understanding Grade Descriptors tool serves several key purposes:

  • Helps students understand the specific criteria used for assessment
  • Provides visual presentation of grade bands with colour-coding
  • Offers targeted advice on how to improve and move up to higher grade bands
  • Aligns with the School of Design Curriculum Framework
  • Creates a common understanding of standards between staff and students

1.2 Tool Components

The tool consists of:

  • An interactive grade band selector with colour-coded buttons
  • Detailed description of assessment criteria for each grade band
  • Specific advice on how to improve performance
  • A comprehensive grading matrix showing all grade bands at once

2. Getting Started

2.1 Required Files

To customise the tool, you'll need:

  • The KSA_Talk_Understanding_Grade_Descriptors.html file
  • A suitable header image for your programme (optional)
  • A text editor (e.g., Notepad++, Visual Studio Code, or even basic Notepad)

2.2 Downloading the Template

You can obtain the HTML template file from:

  1. The GitHub repository: https://github.com/MounseyWood/Understanding-Grade-Descriptors-v4
  2. Your faculty's shared resources folder
  3. By contacting your Canvas Administrator, Associate Professor David Lawrence d.lawrence@kingston.ac.uk

2.3 Opening the File

Open the HTML file in a text editor. Look for the following key sections that you'll need to modify:

  • Header section with title and introduction
  • Grade bands data structure

3.2 Grade Bands Data Structure

The grade bands are defined in the JavaScript code. Look for the grades array in the script section:

const grades = [
  {
    id: 'first-plus',
    title: 'Distinction',
    range: '85% and above',
    color: 'bg-[#80FF80]',
    textColor: 'text-black',
    qualities: [
      // Assessment criteria for this grade
      { title: 'Research & Analysis', description: 'Your tutors are looking for...' },
      // More criteria...
    ]
  },
  // More grade bands...
];

Each grade band has the following properties that you may need to modify:

  • id: A unique identifier (don't change unless necessary)
  • title: The name of the grade band (e.g., "Distinction", "First Class")
  • range: The percentage range (e.g., "85% and above")
  • color: The background colour for this grade band
  • textColor: The text colour for this grade band
  • qualities: An array of assessment criteria objects

Note on Colours: The default colours use a traffic light system from green (highest grades) through amber to red (lowest grades). You may wish to keep this intuitive colour scheme or adapt it to your department's branding.

4. Customising Learning Outcomes and Assessment Criteria

Each grade band contains assessment criteria organised into four categories aligned with the School of Design Curriculum Framework. You'll need to customise these descriptions to match your programme's specific learning outcomes and assessment requirements.

4.1 Assessment Criteria Structure

The criteria are stored in the qualities array for each grade band:

qualities: [
  { 
    title: 'Research & Analysis', 
    description: 'Your tutors are looking for exceptional research that goes far beyond expectations. You should demonstrate original thinking and provide strong evidence to support your ideas. Your work should show deep understanding of relevant contexts and critically analyze sources with comprehensive evidence.' 
  },
  // More criteria...
]

Each criterion has:

  • title: The name of the criterion (e.g., "Research & Analysis")
  • description: A detailed explanation of what is expected at this grade level

4.2 Customising the Criteria Categories

If your programme uses different assessment criteria categories than the default four (Research & Analysis, Experimentation & Practice, Communication & Presentation, Personal & Professional Development), you can change the titles and descriptions to match your programme's framework.

Important: If you change the number of criteria categories, you'll need to make corresponding changes throughout the file, including in the comprehensive grading matrix section.

6. Customising the Grading Matrix

The complete grading matrix provides an overview of all grade bands and criteria in a single table. This appears in a popup when students click the "Complete Grading Matrix" button.

6.1 Matrix Structure

Look for the following section in the HTML file:

<table className="min-w-full">
  <thead>
    <tr>
      <th>Criteria</th>
      {matrixGrades.map((grade) => (
        <th 
          key={grade.id}
          className={`${grade.color} ${grade.textColor}`}
        >
          {grade.title} ({grade.range})
        </th>
      ))}
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>Research & Analysis</th>
      <td>Exceptional levels of research and synthesis. Original and convincing interpretation.</td>
      <td>Extensive research, independent synthesis and evaluation.</td>
      <td>Wide ranging and well-informed research and accurate evaluation.</td>
      <td>Accurate and appropriate research of elements.</td>
      <td>Adequate range from standard sources. May lack accurate interpretation</td>
      <td>Limited source information evidenced.</td>
      <td>Little or no source information evidenced.</td>
    </tr>
    <!-- More rows for other criteria -->
  </tbody>
</table>

Customise each cell in the table with concise descriptions of the criteria for each grade band.

Note: The matrix content should be concise summaries of the full criteria descriptions, as space is limited in the table format.

7. Colour Scheme Customisation

You may wish to customise the colours used for each grade band to match your departmental branding or to make them more accessible.

7.1 Colour Properties

For each grade band in the grades array, you can modify the following properties:

  • color: The background colour (e.g., bg-[#80FF80])
  • textColor: The text colour (e.g., text-black)

7.2 Grade Band Colour System

The tool uses a 'traffic-light' colour system to denote grade bands, from high (green) through amber to low (red). Here are the hex codes:

  • Distinction (85% and above): #80FF80 (bright green)
  • First Class (70-84%): #BFFF80 (light green)
  • Upper Second (60-69%): #FFFF80 (yellow)
  • Lower Second (50-59%): #FFE880 (light amber)
  • Third Class (40-49%): #FFCC80 (amber)

8. Testing and Deployment

8.1 Testing Your Customisations

Before deploying your customised tool, test it thoroughly:

  1. Open the HTML file in a web browser to ensure it displays correctly
  2. Test all interactive elements (grade band buttons, grading matrix popup)
  3. Check that all your customised text appears correctly
  4. Verify that the colours display as expected
  5. Test on different devices (desktop, tablet, mobile) if possible

8.2 Deploying to Canvas VLE

To deploy your customised tool to Canvas:

  1. Log in to your Canvas course
  2. Create a new page or edit an existing page
  3. Click on the HTML Editor button (</>) in the Rich Content Editor
  4. Copy and paste your entire customised HTML file content
  5. Save the page

Note: If you're using a custom header image, you'll need to upload it to Canvas Files first and update the image path in your HTML accordingly.

9. Troubleshooting

9.1 Common Issues

Here are some common issues you might encounter and how to resolve them:

Issue: Page doesn't load or displays incorrectly

Solution: Check for syntax errors in your HTML or JavaScript code. Look for missing closing tags, quotes, or brackets.

Issue: Interactive elements don't work

Solution: Ensure you haven't accidentally removed or modified essential JavaScript functions or event handlers.

Issue: Images don't display

Solution: Verify the image paths are correct and that images have been uploaded to the correct location in your Canvas course.

Issue: Text appears in the wrong language

Solution: Make sure you've updated all text content in the file, including any hidden or less obvious sections.

9.2 Getting Help

If you encounter issues you can't resolve:

  • Consult your faculty's digital learning team or e-learning specialists
  • Contact your Canvas Administrator, Associate Professor David Lawrence d.lawrence@kingston.ac.uk for assistance with the Understanding Grade Descriptors tool specifically
  • Refer to the GitHub repository for the latest version and any additional documentation

10. Conclusion

By customising the Understanding Grade Descriptors tool for your programme, you can provide your students with a clear, interactive guide to assessment criteria that helps them understand what is expected at each grade level and how they can improve their performance.

Remember that the tool should be reviewed and updated regularly to ensure it remains aligned with your programme's assessment criteria and pedagogical approach.

We hope this guide has been helpful. If you have suggestions for improving either the tool or this guide, please contact your Canvas Administrator, Associate Professor David Lawrence d.lawrence@kingston.ac.uk.
